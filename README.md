# AI News Digest Skill

一个面向 **AI 最新资讯追踪** 的 Codex Skill。

它不是简单的“新闻抓取脚本”，而是一套从 **高可信信源筛选**、到 **中文日报生成**、再到 **网页化归档发布** 的完整工作流。

## 为什么这个 Skill 值得用

每天的 AI 新闻很多，但真正值得追踪的信息其实很少。

这个 Skill 的目标，不是把互联网上所有 AI 内容都堆给你，而是从一手和高可信来源中，尽可能快速地筛出 **真正有信号的更新**，并用统一、可读、适合归档的中文格式输出出来。

它特别适合这些场景：

- 每日 AI 情报追踪
- AI 行业日报 / 周报
- 公司内部情报简报
- 模型、平台、安全、政策、产业动态监测
- 面向中文读者的 AI 信息整理与发布

## 核心能力

- 获取最新 AI 资讯，优先官方博客、新闻室和公开高可信 feed
- 识别“今天 / 最新 / 最近 / 本周”等时间范围，并转成精确日期
- 去重、筛选、排序，只保留高信号动态
- 默认输出全中文 Markdown 日报
- 自动生成单页 HTML 页面
- 自动维护按日期浏览的 `index.html` 聚合页
- 支持快速草稿模式，也支持更强的中文精修模式

## 我最重视的部分：信源

这个 Skill 的价值，首先来自 **信源质量**。

它优先使用：

- 官方博客
- 公司新闻室
- 研究机构公开更新
- 公开 RSS feed
- 少量可信科技媒体 feed 作为补充发现渠道

当前一键生成器默认接入的公开源包括：

- Google Blog
- Microsoft Blog
- NVIDIA Blog
- Meta Newsroom
- TechCrunch AI

这意味着它默认更偏向：

- 官方公告
- 平台更新
- 产品发布
- 安全与治理动态
- 研究与产业落地进展

而不是泛泛的二手搬运、SEO 内容农场，或者没有明确出处的社交媒体热帖。

## 输出长什么样

默认输出为 **全中文结构化日报**，包含：

- 今日要点
- 重点动态
- 行业观察
- 接下来值得关注

并支持进一步渲染为：

- 单页 HTML 日报
- 多日报归档页 `index.html`

这让它不仅能“整理信息”，还能直接形成一套可持续积累的内容资产。

## 使用方式

### 1. 一键生成日报草稿

```bash
python3 scripts/generate_report.py --date YYYY-MM-DD --reports-dir reports
```

### 2. 使用更强的中文精修模式

```bash
python3 scripts/generate_report.py --date YYYY-MM-DD --reports-dir reports --review
```

### 3. 从已有 Markdown 发布 HTML 和归档页

```bash
python3 scripts/publish_report.py --date YYYY-MM-DD --reports-dir reports
```

## 这个仓库里有什么

- `SKILL.md`：Skill 主说明与触发逻辑
- `references/`：来源策略和日报模板
- `scripts/generate_report.py`：自动抓取并生成中文日报草稿
- `scripts/publish_report.py`：把 Markdown 发布为 HTML 并更新归档页
- `scripts/build_report_index.py`：重建日报聚合页

## 当前边界

这个 Skill 已经适合日常使用，但我不会把它包装成“全自动编辑部”。

它目前的真实定位是：

- 一个高质量的 AI 中文日报生成器
- 一个强调信源质量和结构化输出的工作流
- 一个适合持续迭代和扩展的数据源与发布框架

它已经能做到：

- 自动生成高质量草稿
- 自动发布页面
- 自动维护归档

但如果你要做正式对外发布，仍然建议做一次快速人工审阅，尤其是：

- 标题自然度
- 重要性排序
- 是否需要补充第二来源确认

## 仓库目标

这个项目的目标，不是做“信息越多越好”的 AI 新闻爬虫。

而是做一个更克制、更可信、对中文读者更友好的 **AI 资讯生产工具**：

- 有信源
- 有结构
- 有节奏
- 能归档
- 能持续迭代
